PrimeNumbersAPI

Getting the 1000 first prime numbers with NodeJS.

What is PrimeNumbersAPI?

It's a simple webservice that returns the 1000 first prime numbers. You can use this API to do things related to searches, tests, Math operations, etc. Prime numbers, by the way, are natural numbers greater than one that have only two divisors: 1 and themselves. 2, 3, 5, 11, 29, for example, are prime numbers.

The API were written in JavaScript and uses NodeJS as runtime. The values are returned in JSON-array format.

Github repository link: https://github.com/gustavoveloper/prime-numbers-api.

Response format

The API is JSON. It means that all HTTP responses are sent in JSON format.
By the way, the response body have the same format for all requests: it has an error and an set of data. { "error": <response-error>, "data": <response-data> }

If any error occurs, the data field has null as value and the error field contains the error message. Else, the error field receives null and the data field contains the response data.

Using the API

The API has two endpoints: one for retrieving prime numbers and one for accessing this documentation. Let's see each one of them.
The used HTTP user-agent will be curl.

Retrieving the first 1000 prime numbers

With curl, make a GET request to the /api endpoint: $ curl http://host:8080/api OBS.: In this webservice, all route names are case-insentitive.
Your result must be something like: HTTP/1.1 200 OK
Server: PrimeNumbersAPI-Server
Date: Sun, 10 Jul 2022 20:27:41 GMT
<HTTP headers...>

{ "error": null, "data": [ 2, 3, 5, 7, 11, 13, 17, 19, ... ] } After sending the response, the API server closes the connection with the client by default (the Content-Type HTTP header is set to close for all responses).

The /docs endpoint - Accessing the documentation

If you access the /docs API endpoint using a browser, you will be redirected to this site, that contains the documentation.
